# mdl-repeat

**MDL-guided seed-and-extend de novo repeat element discovery.**

mdl-repeat identifies repetitive element families in genomic DNA sequences without prior knowledge of a repeat library. It faithfully replicates the RepeatScout v1.0.7 discovery engine (N-sequence simultaneous banded DP extension) and replaces arbitrary thresholds with the **Minimum Description Length (MDL) principle** — an information-theoretic criterion that automatically determines how many families to report, the minimum consensus length, and the minimum copy number that justifies a family.

## Features

- **MDL-based model selection** — replaces `MINTHRESH`, `GOODLENGTH`, and all other hard-coded thresholds with a unified, principled criterion
- **RepeatScout-compatible discovery** — same seed-and-extend algorithm, same l-mer frequency table format, same alignment scoring
- **Refinement pipeline** — merge redundant families (80-80-80 rule), split bimodal divergence, assemble TE fragments, prune marginal families
- **Three MDL position encoding modes** — `exact` (lgamma binomial, default), `upper` (conservative bound), `none` (backward compatibility)
- **Multi-threading** — parallel k-mer position index construction and alignment refinement
- **Frequency table reuse** — save/load l-mer frequency tables to skip recomputation across runs

## Requirements

- GCC (C11 standard)
- GNU Make
- Linux/POSIX
- No external libraries (only `libm` and pthreads)

## Quick Start

```bash
# Single-step: discover and output repeat library
./bin/mdl-repeat -sequence genome.fa -output families.fa -v

# Two-step: save l-mer table for reuse
./bin/mdl-repeat -sequence genome.fa -output families.fa -freq-output genome.freq
./bin/mdl-repeat -sequence genome.fa -output families.fa -freq genome.freq

# Full output (library + instance locations + statistics)
./bin/mdl-repeat -sequence genome.fa \
    -output families.fa \
    -instances instances.bed \
    -stats stats.tsv \
    -threads 4 -vv
```

**RepeatScout compatibility**: mdl-repeat can read frequency tables generated by RepeatScout's `build_lmer_table`:

```bash
# Use RepeatScout's pre-computed frequency table
build_lmer_table -sequence genome.fa -freq genome.freq -l 14
./bin/mdl-repeat -sequence genome.fa -output families.fa -freq genome.freq -l 14
```

## Usage

```
mdl-repeat -sequence <file> -output <file> [options]

Required:
  -sequence <file>   Input FASTA (single or multi-sequence)
  -output <file>     Output repeat library (FASTA)

Discovery:
  -freq <file>       Pre-computed l-mer frequency table
  -freq-output <file> Write l-mer frequency table for reuse
  -l #               L-mer length (default: auto = ceil(1+log4(N)))
  -L #               Max extension distance per side (default: 10000)
  -minthresh #       Min l-mer frequency to seed (default: 2)
  -goodlength #      Min consensus length pre-filter (default: 30)
  -maxgap #          Max DP band offset (default: 5)
  -match #           Match score (default: 1)
  -mismatch #        Mismatch score (default: -1)
  -gap #             Gap penalty (default: -5)
  -cappenalty #      Cap penalty (default: -20)
  -minimprovement #  Min improvement per step (default: 3)
  -stopafter #       Stop after N no-progress positions (default: 100)
  -maxentropy #      Entropy filter threshold (default: -0.70)
  -tandemdist #      Min distance between same-strand l-mers (default: 500)
  -maxoccurrences #  Max occurrences per seed (default: 10000)
  -maxrepeats #      Max families to discover (default: 100000)

Refinement:
  -threads #         Number of threads (default: 1)
  -mdl-mode <mode>   MDL position encoding: none|exact|upper (default: exact)

Output:
  -instances <file>  Instance locations (BED6)
  -stats <file>      Per-family statistics (TSV)
  -v / -vv           Verbosity level
```

## Pipeline

```
Input FASTA
    |
    v
1. Load genome (pad + boundary tracking)
    |
    v
2. Discover consensus families (seed-and-extend)
   - Build/read l-mer frequency table
   - For each high-frequency seed: N-sequence simultaneous banded DP extension
   - Mask found families to prevent rediscovery
    |
    v
3. Build k-mer table + position index (for refinement)
    |
    v
4. Compact (remove dead/short families)
    |
    v
5. Merge redundant families (80-80-80 rule + union-find)
    |
    v
6. Split bimodal families (Otsu's method, MDL-validated)
    |
    v
7. Assemble TE fragments (spatial co-occurrence, nesting guard)
    |
    v
8. MDL scoring and library selection (iterative R convergence)
    |
    v
9. Prune marginal families (exclusive coverage vs. model cost)
    |
    v
10. Post-prune recovery pass (re-score with reduced R)
    |
    v
Output: FASTA library + BED instances + TSV statistics
```

## Output Formats

**FASTA** (`-output`): Consensus sequences for accepted repeat families.

```
>R=0 length=312 copies=15 mdl=1204.3
ACGTACGT...
```

**BED6** (`-instances`): Genomic coordinates of every instance.

```
chr1    10432    10744    R=0    850    +
```

Column 5 is `1000 * (1 - divergence)` on a 0-1000 scale.

**TSV** (`-stats`): Per-family summary (consensus length, copy count, mean divergence, MDL score, model cost, topology).

## How It Works

### MDL Model Selection

The Minimum Description Length principle asks: does adding a repeat family to the library reduce the total cost of describing the genome?

```
DL(Genome) = DL(Library) + DL(Genome | Library)
```

- **Library cost**: `L_int(R) + sum(L_int(len_r) + 2 * len_r)` — encoding R consensus sequences at 2 bits/base
- **Per-instance savings**: `2 * aligned_length - instance_encoding_cost` — savings vs. literal encoding
- **MDL score**: `total_savings - model_cost` — a family is accepted iff this is positive

This replaces RepeatScout's arbitrary thresholds with a single, principled criterion. No minimum copy count or minimum length is needed — the encoding scheme inherently penalizes noise and rewards genuine repeats.

### Seed-and-Extend Discovery

Faithfully replicates RepeatScout v1.0.7 with two bug fixes:
1. **Sequence boundary check** — RepeatScout hard-codes `bStart=0; bEnd=50000000`; mdl-repeat computes from actual input
2. **Dynamic value[] allocation** — RepeatScout overflows when l > 19; mdl-repeat allocates dynamically

### Refinement

- **Merge**: 80-80-80 rule (Wicker et al. 2007) with k-mer Jaccard pre-screening and union-find transitive merging
- **Split**: Otsu's method on divergence histograms, MDL-validated
- **Fragment assembly**: Spatial co-occurrence sweep-line detects adjacent TE fragments; nesting guard prevents merging nested elements
- **Prune**: Exclusive coverage test removes families whose unique contribution doesn't justify their model cost
- **Recovery**: Post-prune re-scoring recovers families that become viable when R decreases

## Architecture

```
src/
├── main.c              Pipeline driver, CLI parsing
├── types.h             Core types (gpos_t, glen_t, DNA encoding, constants)
├── genome.c/.h         FASTA loading, padding, boundary tracking
├── kmer.c/.h           Canonical k-mer counting, hash table, position index
├── discover.c/.h       RepeatScout-style seed-and-extend engine (~2200 lines)
├── align.c/.h          Multi-k-mer seeding, banded DP alignment, consensus rebuild
├── candidates.c/.h     CandidateFamily/Instance structures, memory management
├── refine.c/.h         Merge, split, fragment assembly, prune (~1800 lines)
├── mdl.c/.h            Rissanen's log* code, MDL scoring, library selection
├── output.c/.h         FASTA, BED6, TSV writers
└── cmd_line_opts.c/.h  Argument parsing (from RepeatScout)
```

~6,000 lines of C. No external dependencies.

## Key Differences from RepeatScout

| | RepeatScout | mdl-repeat |
|---|---|---|
| Model selection | Hard thresholds (MINTHRESH=3, GOODLENGTH=50) | MDL principle (no thresholds) |
| Library quality | Raw output, needs post-filtering | Merge + split + assembly + prune |
| Fragment handling | None | Spatial co-occurrence assembly |
| Subfamily detection | None | Otsu-based divergence splitting |
| Default min frequency | 3 | 2 (MDL decides) |
